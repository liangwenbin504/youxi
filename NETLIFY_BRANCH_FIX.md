# Netlify部署步骤 - 分支选择问题解决

## 当前问题：分支下拉菜单为空

你在Netlify部署配置页面遇到了分支下拉菜单为空的问题。这可能是因为Netlify需要一点时间来同步仓库的分支信息，或者是刚创建的仓库分支信息还没有完全同步。

## 解决方法

### 方法1：刷新页面
- 按下浏览器的刷新按钮（F5）
- 等待页面重新加载
- 检查分支下拉菜单是否显示了分支选项

### 方法2：手动输入分支名称
- GitHub现在的默认分支是 `main`
- 尝试直接在「Branch to deploy」输入框中**手动输入 `main`**
- 输入完成后，点击页面底部的「Deploy site」按钮

### 方法3：检查GitHub仓库
- 打开浏览器，访问 `https://github.com/liangwenbin504/youxi`
- 查看仓库的默认分支是什么（通常是 `main`）
- 然后回到Netlify页面，手动输入该分支名称

## 其他配置项
- **Base directory**: 留空
- **Build command**: 留空（静态网站不需要构建）
- **Publish directory**: 留空（HTML文件在根目录）

## 关键点
- 刚创建的仓库可能需要时间同步分支信息
- 可以尝试手动输入分支名称 `main`
- 静态网站不需要复杂的构建配置

现在尝试手动输入 `main` 作为分支名称，然后点击「Deploy site」按钮开始部署吧！
